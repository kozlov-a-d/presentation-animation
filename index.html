CSS анимации

Просты

Выполняются вне основного потока

Синхронизированы с частотой обновления экрана

Слабый контроль

Пример: transition


  .element{
    transition: transform 1s ease-in;
  }

  .element:hover{
    transform: rotate(360deg) translateY(40px);
  }
Hover Me

Свойство transition состоит из:

transition-property — указываем какие css свойства нужно анимировать ( можно all )

transition-duration — продолжительность анимации

transition-timing-function — временная функция

transition-delay — задержка перед началом анимации

Пример: animation (@keyframe)


  .element:hover{
    animation: animtaion-rotate 1s ease-in;
  }

  @keyframe animtaion-rotate {
    from { transform: rotate(0deg); }
    50% { transform: rotate(360deg); }
    to { transform: rotate(0deg); }
  }
Hover Me

Свойство animation состоит из:

animation-name — задаёт @keyframes, настраивающий кадры анимации

animation-delay — задержка перед началом анимации

animation-iteration-count — количество повторенией ( infinite для бесконечного повторения анимации )

animation-duration — продолжительность цикла анимации

animation-direction — направление анимации

animation-timing-function — временная функция

animation-fill-mode — значения, используемые анимацией, до и после исполнения

animation-play-state — позволяе приостановить и возобновить анимацию.

Как работает рендеринг в браузере?





JavaScript — какой либо js код ( работа с данными, добавление DOM-элементов на страницу и тд)

Style — вычисление стилей, какие правила CSS к каким элементам применяются

Layout — браузер вычисляет, сколько каждый элементу займет места, и где он находится на экране

Paint — подразумевает вывод текста, цветов, изображений, границ и теней, по сути – всех визуальных частей элементов. Прорисовка обычно выполняется на нескольких поверхностях, которые называются слоями.

Composite — т.к. части страницы были отрисованы на нескольких слоях, их нужно собрать вместе

Как облегчить браузеру жизнь?



CSS Triggers — Paul Lewis

Gist "What forces layout / reflow" — Paul Irish



Если изменить свойство, которое связано только с прорисовкой, например фоновое изображение, цвет текста или его тень, другими словами, свойство, которое не влияет на макет страницы, браузер оставит макет неизменным, но ему все равно придется выполнить прорисовку.

Если же изменить свойство, которое не требует ни перерасчета макета, ни прорисовки, браузер сразу же перейдет к компоновке.
